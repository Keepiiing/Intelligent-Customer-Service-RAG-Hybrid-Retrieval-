# 智能客服系统（RAG方向）工作流程文档

## 1. 文档目的

本文档用于描述智能客服系统从需求启动到上线迭代的完整工作流程，明确阶段目标、关键动作、角色分工与运行闭环，便于项目实施、团队协作和后续复盘。

## 2. 总体流程

项目按五个阶段推进：

1. 项目启动与需求拆解
2. 知识库构建
3. 核心模块开发
4. 系统集成与部署
5. 测试上线与持续优化

## 3. 分阶段工作流程

### 3.1 阶段一：项目启动与需求拆解

#### 输入

- 业务目标
- 客服问题样本
- 合规要求
- 知识来源清单

#### 核心动作

1. 联合产品、业务、合规团队梳理核心需求。
2. 明确知识源范围、用户痛点、性能指标和合规红线。
3. 输出整体架构方案和技术选型方案。
4. 搭建 PoC，验证解析、检索和推理链路可行性。

#### 输出

- 需求文档
- 技术架构图
- PoC 验证结论
- 阶段实施计划

### 3.2 阶段二：知识库构建

#### 输入

- 原始文档
- 多语言资料
- 表格数据
- 历史客服语料

#### 核心动作

1. 用解析工具抽取 PDF、Word、Excel、扫描件内容。
2. 对解析结果进行清洗、去重、格式统一和缺失补齐。
3. 抽取金融领域实体与关系。
4. 将文本向量化后写入 Milvus。
5. 将关系数据写入 Neo4j。
6. 建立增量更新、版本控制和生效机制。

#### 输出

- 可检索知识库
- 知识图谱
- 向量索引
- 增量更新流程

### 3.3 阶段三：核心模块开发

#### 3.3.1 检索模块流程

1. 接收用户问题。
2. 执行关键词匹配、意图识别和语义分析。
3. 根据问题类型选择基础路径、混合路径或增强路径。
4. 执行 BM25、向量检索、图谱检索或 Query Rewriting。
5. 对召回结果进行 Reranker 精排。
6. 根据阈值判断是否触发兜底检索。

#### 3.3.2 生成模块流程

1. 将精排后的上下文输入生成链。
2. 按 Prompt 模板生成结构化回答。
3. 在多轮场景下追加历史上下文。
4. 在需要时调用外部工具获取实时数据。
5. 控制回答长度、格式和语言风格。

#### 3.3.3 合规模块流程

1. 输入前进行敏感信息识别与过滤。
2. 检索结果进入规则引擎做中间校验。
3. 生成结果做风险识别、敏感内容拦截和来源校验。
4. 高风险答案触发人工审核或转人工客服。

#### 输出

- 可运行的检索服务
- 可运行的生成服务
- 合规校验能力
- 统一问答链路

### 3.4 阶段四：系统集成与部署

#### 核心动作

1. 使用 FastAPI 封装统一对外接口。
2. 串联知识库、检索、生成、合规模块。
3. 通过 Docker 实现容器化部署。
4. 结合 K8s、Nginx 或网关完成负载均衡与扩缩容。
5. 配置 LangFuse、Grafana 等监控能力。
6. 打通 CI/CD、测试环境、灰度发布和回滚机制。

#### 标准链路

用户请求进入系统后，按以下顺序执行：

1. 输入预处理
2. 意图识别
3. 检索路由
4. 检索执行
5. 精排筛选
6. 合规校验
7. 答案生成
8. 输出过滤
9. 结果返回
10. 日志与指标上报

### 3.5 阶段五：测试上线与持续优化

#### 上线前

1. 执行功能测试，验证多场景问答链路。
2. 执行性能测试，模拟高并发访问。
3. 执行合规测试，验证敏感信息、风险回答和来源追溯。
4. 与业务、合规团队共同验收。

#### 上线后

1. 采集用户满意度、自动化处理率、检索和生成效果数据。
2. 基于 LangFuse 和 Grafana 定位异常。
3. 对召回率、精排效果、Prompt、模型参数进行迭代。
4. 对新增知识、低满意度案例和异常日志进行复盘。
5. 形成“监控发现问题、实验验证方案、发布上线、效果回看”的优化闭环。

## 4. 用户侧业务流程

1. 用户通过 APP、网页或 API 发起咨询。
2. 系统识别输入类型，语音转文本并识别语言。
3. 系统执行敏感信息初筛。
4. 系统选择对应检索路径并完成检索与推理。
5. 系统输出带来源依据的结构化回答。
6. 用户可继续追问或提交满意度反馈。
7. 若无法自动处理，系统自动转人工并同步上下文。

## 5. 团队协作流程

1. 业务、客服、合规团队提出需求或问题。
2. 产品与算法团队完成方案拆解。
3. 算法团队负责知识库、检索、生成、评估优化。
4. 工程团队负责接口、部署、监控、发布。
5. 测试团队负责功能、性能、合规验证。
6. 合规团队负责审核和风险把控。
7. 上线后一周内进行联合复盘，确定下一轮优化项。

## 6. 异常处理流程

### 6.1 回答不准确

1. 优先排查检索召回是否不足。
2. 再排查精排是否漏掉高相关内容。
3. 然后排查 Prompt 和模型生成质量。
4. 最后回看知识库内容是否缺失或解析错误。

### 6.2 敏感信息泄露

1. 立即下架相关自动回复并切人工。
2. 通过日志和追踪定位泄露来源。
3. 修复知识库脱敏、Prompt 约束和输出过滤规则。
4. 补充测试集并执行专项审计。

### 6.3 响应时延升高

1. 先看 CPU、显存、检索时延、生成时延、队列长度。
2. 检索瓶颈则处理缓存、索引和向量库扩容。
3. 生成瓶颈则调整 batch size、量化、实例数。
4. 接口瓶颈则优化连接池、协议和限流策略。

## 7. 关键交付物

- 需求文档
- 技术架构与流程图
- 知识库构建方案
- 检索与生成服务
- 合规模块
- 部署方案与 CI/CD 配置
- 监控告警方案
- 测试报告与上线复盘材料

## 8. 运行闭环

系统长期运行遵循以下闭环：

1. 知识持续更新
2. 用户问题持续进入
3. 检索与生成效果持续监控
4. 异常与低质量回答持续回收
5. 策略、模型、Prompt 持续优化
6. 指标回看驱动下一轮迭代
